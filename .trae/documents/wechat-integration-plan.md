# 微信接入开发规划

最终目标：让「灵伴」AI 智能体能够在微信里与用户正常聊天，并和网页端数据互通。

---

## 一阶段：完善基础产品与体验

**目标**：把现有网页端打磨稳定，为后续微信接入打好基础。

1. 用户体系
   - 匿名登录与用户持久化；
   - 用户头像、昵称、人设可设置；
   - 用户人设用于和智能体对话时的系统上下文。

2. 智能体功能
   - 智能体的创建、编辑、删除；
   - 头像、人设、模型、API Key 配置；
   - 多轮对话历史保存。

3. 视觉与交互
   - 主题色切换（极光渐变、七彩渐变）；
   - 移动端优先的响应式布局；
   - 空状态、加载态、错误提示优化。

4. 稳定性
   - Render 部署与 PostgreSQL 迁移稳定运行；
   - 构建、类型检查、Lint 全部通过；
   - 基础日志与健康检查。

---

## 二阶段：多端同步与账号体系

**目标**：让用户在任何终端都能获得一致的对话体验。

1. 账号打通
   - 支持 Token + 设备绑定；
   - 网页端可扫码或授权绑定微信 OpenID；
   - 同一用户在网页和微信共用同一套会话数据。

2. 消息同步
   - 聊天记录按 `userId + agentId` 统一存储；
   - 新消息实时同步到所有在线终端；
   - 离线消息在用户重新进入时补齐。

3. API 规范化
   - RESTful API 版本管理；
   - 统一错误码与返回格式；
   - 为微信端预留独立接口入口。

4. 数据扩展
   - 增加消息来源字段（web / wechat）；
   - 增加智能体开关（是否允许微信访问）；
   - 增加微信用户表（openid、unionid、绑定关系）。

---

## 三阶段：微信生态准备

**目标**：完成微信侧账号与服务器准备。

1. 账号注册
   - 注册微信公众号/小程序并完成认证；
   - 获取 AppID、AppSecret；
   - 配置业务域名、服务器域名、白名单。

2. 服务器配置
   - Render 服务配置公网 HTTPS 入口；
   - 配置微信服务器 URL、Token、EncodingAESKey；
   - 完成微信服务器签名验证。

3. 安全与合规
   - 消息加解密；
   - access_token 集中缓存与定时刷新；
   - 隐私政策与内容安全说明。

4. 基础接口预研
   - 微信消息 XML 解析；
   - 被动回复与客服消息接口；
   - 素材上传与下载接口。

---

## 四阶段：接入微信正常聊天

**目标**：实现微信端与智能体的正常对话。

1. 消息接收
   - 开发 `POST /api/wechat/:agentId` 接收微信推送；
   - 校验签名、解析 XML、提取 OpenID 与消息内容；
   - 根据 OpenID 找到或创建本地用户。

2. 消息回复
   - 将微信消息写入 `ChatMessage`；
   - 复用现有 LLM 对话逻辑生成回复；
   - 5 秒内被动回复简单文本，复杂回复通过客服消息异步推送。

3. 多轮对话
   - 微信端历史记录与网页端共享；
   - 用户人设与智能体人设共同生效；
   - 超过 48 小时未互动时提示用户重新发起。

4. 入口与路由
   - 每个智能体对应独立微信入口 URL；
   - 支持通过菜单/关键词切换智能体；
   - 网页端可控制智能体是否对外暴露。

---

## 后续：优化用户体验和其他

**目标**：持续打磨，让产品更稳定、更好用。

1. 体验优化
   - 微信端回复速度优化；
   - 长文本自动拆分；
   - 图片、语音、视频消息支持；
   - 更丰富的卡片式回复。

2. 稳定性优化
   - 微信消息队列与幂等去重；
   - LLM 超时与失败兜底；
   - 监控、告警、日志聚合。

3. 功能扩展
   - 微信群聊接入；
   - 小程序端完整 UI；
   - 企业微信接入；
   - 分享、邀请、统计。

4. 商业化与其他
   - 用户增长与留存分析；
   - 付费方案与用量控制；
   - 多语言、暗黑/亮色模式；
   - 开放 API 与插件生态。
